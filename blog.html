Important FAQ

What is the Document Object Model (DOM)?

In a webpage, every element is treated as an object. If we think about JavaScript object, there are properties, methods or nested objects in an object. Like JavaScript object, we can access an element from webpage using its different property and method. For example, we can access the 'text' of a 'p' tag using JavaScript with the property name 'innerText'. We can also access a child of a parent element just like a nested object. So DOM is an organized hierarchy of elements inside a webpage so that we can easily access and interact with them.

How do you select an element from the DOM?

A DOM element can be selected in different ways. We can use HTML class, id, tag etc. It is described below -

getElementById: It is a way of selecting a specific HTML element by its id. It is used to select a single HTML element.
E.g. document.getElementById("demo");

getElementsByClassName: It is a way of selecting multiple elements with same class name. It returns a 'HTMLCollection' containing multiple elements.
E.g. document.getElementsByClassName("demo");

getElementsByTagName: It is also a way of selecting multiple elements. It selects multiple elements with same tag.
E.g. document.getElmentsByTagName("img");

querySelector: It is a way of selecting specific element using CSS selector.
E.g. document.querySelector(".demo");
Note: If there are more than one element with same identifier, querySelector will select the first one.

querySelectorAll: It is a way of selecting multiple elements using CSS selector.
E.g. document.querySelectorAll(".demo");

What is event delegation in the context of DOM and why is it useful?

Event delegation is a technique used to handle multiple child elements with a single event listener to a common parent.

Importance of event delegation: Event delegation is useful as it reduces the number of event listener. So we can use only one event listener to handle multiple child element. Suppose there is a 'div' containing three 'buttons'. If an event listener is attached with the 'div', we can encounter any of the three 'button' elements exculsively with the 'target' property when the event is triggered. Without event delegation, we'll have to write an event listener for each button. It is unimaginable to write an event for each element if there are ten or twenty elements. But we don't have be worried as we have event delegation to save us. No matter how many children a parent element contains, event delegation can handle them with just one event listener.

How do you manipulate an element's attributes and styles using the DOM?

There are bunch of ways to manipulate an element's attributes and styles using the DOM.

setAttribute & getAttribute: The 'setAttribute' method is used to set attribute to a DOM element. It has two arguments. First contains attribute name and the second is for attribute value.
E.g. document.getElmentById("demo").setAttribute("title", "Hello World");
The getAttribute returns the attribute value of a DOM element.
E.g. document.getElmentById("demo").getAttribute("title");
It'll return 'null' if the given attribute isn't present in the element.

className & classList: These two are used to modify the classes of a DOM element. 'className' is used to modify completely the value of class attribute. On the other side, classList is used to add, remove or toggle a single class of a DOM element.
E.g. document.getElmentById("demo").className = "";
document.getElmentById("demo").classList.add("hello");

Using css properties: We can modify and add new inline css to a DOM element.
E.g. document.getElmentById("demo").style.backgroundColor = "red";
document.getElmentById("demo").style.width = "250px";
